# 📦 Inventory Automation: Operational Logic Engine

Welcome to the core feature directory of this repository. This folder hosts a functional data ingestion pipeline designed to bridge the gap between supply chain mechanics and software architecture by automating raw operational data processing.

### 🧠 Core Learning Objectives

- **Data Ingestion Pipelines:** Reading, cleaning, and processing raw text-based formats (CSV) into memory.
- **Financial Metric Calculations:** Automating unit valuation and categorical grouping analytics.
- **Robust Error Handling:** Utilizing safety catch-blocks to manage missing files or mismatched input structures.

## 📂 Included Course Projects

### 🔹 1. **Data Analytics Pipeline (parse_metrics.py)**

- **Language:** Python 🐍
- **Concepts Demonstrated:** Object-Oriented Principles, File I/O Streams, Structured Loop Parsing.
- **Description:** A robust utility engine that ingests bulk operational records, sanitizes numeric values, and calculates complex stock values automatically.

### 🔹 2. **Mock Supply Chain Data (inventory_manifest.csv)**

- **Language:** CSV Data 📋
- **Concepts Demonstrated:** Structured data design and testing schemas.
- **Description:** A simulated enterprise warehouse matrix containing inventory item indices, counts, and wholesale unit costs.

## 🚀 How to Run These Projects

1. Navigate into the Inventory-Automation directory.
2. Run the main processing logic script using your command line terminal:
   ```bash
   python parse_metrics.py
   ```

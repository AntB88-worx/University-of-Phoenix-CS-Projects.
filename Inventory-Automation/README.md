This folder serves as an independent engineering portfolio showcasing how I leverage diverse software architectures, data pipelines, and object-oriented designs to streamline enterprise supply chains, automate warehouse workflows, and eliminate operational bottlenecks.

* **Data-Driven Efficiency:** Replacing fragile, manual tracking sheets with automated Python extraction and data pipelines.

---

### 🛠️ Portfolio Projects & Design Decisions

#### 🐍 1. Object-Oriented Inventory Pipeline Engine (`inventory_manager.py`)
* **The Architecture:** Advanced Object-Oriented Programming (OOP) Pipeline.
* **The Solution:** A decoupled inventory control engine that translates flat database rows into strongly-typed objects. It isolates structural reporting layers away from raw business data, encapsulating automated financial evaluations via dynamic `@property` methods.
* **Design Rationale:** Engineered to demonstrate true *Encapsulation* and *Separation of Concerns (SoC)*, making the system effortlessly scalable if swapping input formats from a mock payload to cloud APIs or SQL tables.
* **Tech Used:** `Python 3.x`, `OOP Paradigms`, `Data Pipelines`, `Dynamic Encapsulation`
* **Status:**  *Production Ready / Optimized*

#### 🐍 2. High-Efficiency Metrics Parsing Script (`parse_metrics.py`)
* **The Architecture:** Lightweight Procedural Scripting Framework.
* **The Solution:** A high-speed utility script built to instantly stream, parse, and categorize multi-category raw inventory spreadsheets, spitting out structured operational summaries while compiling out-stream alert files (`restock_report.txt`).
* **Design Rationale:** Deliberately built procedurally to bypass object allocation overhead. This showcases the engineering maturity to recognize when a fast, lean script is vastly superior to over-engineering a heavy object framework.
* **Tech Used:** `Python 3.x`, `Procedural Scripting`, `Streaming File I/O`, `Defensive Programming`
* **Status:**  *Production Ready / Optimized*

#### ☕ 3. Enterprise Order Fulfillment State Machine (`FulfillmentTracker.java`)
* **The Architecture:** Strict Enterprise Object-Oriented Design.
* **The Solution:** A robust tracking application mapping physical warehouse workflows directly to data properties. It features a strict regex validation layer at initialization and an internal state sequence enforcer to prevent invalid operational tracking milestones.
* **Design Rationale:** Leverages private variable scoping to guarantee that internal asset states can never be corrupted by arbitrary external scripts, protecting full transactional workflow integrity.
* **Tech Used:** `Java 17+`, `Data Sanitization (Regex)`, `State Engines`, `Encapsulation Scoping`
* **Status:**  *Production Ready / Optimized*
